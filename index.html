About

I design and build infrastructure systems for cross-organizational data exchange in agriculture — platforms that make shared data verifiable, traceable, and trustworthy without a central authority.

My primary project, DEMETER, is a full-stack blockchain-based platform (Hyperledger Fabric + Go + Node.js + Next.js) that records file provenance across organizations with smart contracts, REST APIs, a natural language interface, and DID-based authentication.

I also built AI-based computer vision systems for marine aquaculture, automating feeding decisions and size estimation using YOLO, SAM, and OpenCV.

Data Infrastructure

DEMETER: Data Sharing and Verification Platform

2022 – 2026 · University of Florida · Hyperledger Fabric · Go · Node.js · Next.js · LLM

Overview

DEMETER is a data infrastructure platform that lets independent organizations share files while keeping a verifiable, traceable history on a shared blockchain. Files stay on each organization's local system. Only fingerprints, provenance declarations, and access decisions are recorded on-chain.

I designed and built the full system: chaincode contracts, backend API, off-chain data stores, natural language interface, web frontend, local agent, and deployment pipeline. The working prototype supports 7 operations and trace queries over 8,000 linked records within 5 seconds.

Architecture

DV — Data Verification

Establishes origin identity of a file. SHA-256 fingerprint anchoring on-chain, provider identity binding via DID, Creative Commons license declaration (7 types), visibility control (public/conditional/private), dispute mechanism.

TI — Transformation Integrity

Tracks how files change through processing stages. Transformation/integration declaration, recursive graph traversal, PASS/FAIL integrity check per source.

SI — Sharing Infrastructure

Governs how files move between organizations. Access request workflow (Request → Approve → Fulfill → Download), on-chain transfer records, reuse evaluation, expiration enforcement.
