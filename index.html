About

I design and build infrastructure systems for cross-organizational data exchange in agriculture — platforms that make shared data verifiable, traceable, and trustworthy without a central authority.

My primary project, DEMETER, is a full-stack blockchain-based platform (Hyperledger Fabric + Go + Node.js + Next.js) that records file provenance across organizations with smart contracts, REST APIs, a natural language interface, and DID-based authentication.

I also built AI-based computer vision systems for marine aquaculture, automating feeding decisions and size estimation using YOLO, SAM, and OpenCV.

I am looking for infrastructure engineering roles where I can build systems that solve real data problems at scale.

Projects

Data Infrastructure

1 DEMETER: Data Sharing and Verification Platform 2022–2026 · Hyperledger Fabric · Go · Node.js · Next.js · LLM

A general-purpose data infrastructure platform that lets independent organizations share files while keeping a verifiable, traceable history on a shared blockchain. Files stay on each organization's local system. Only fingerprints, provenance declarations, and access decisions are recorded on-chain.

I designed and built the full system: chaincode contracts, backend API, off-chain data stores, natural language interface, web frontend, local agent, and deployment pipeline. The working prototype supports 7 operations and trace queries over 8,000 linked records within 5 seconds.

2 Natural Language Query Interface 2025 · Phi-3 · Ollama · Node.js

Agricultural data sharing does not require deep reasoning — it requires accurate intent classification and routing. 15 intents with deterministic keyword routing and Phi-3 (3.8B) LLM fallback via Ollama. Running locally with no external API dependency. 74-case evaluation suite with 100% routing accuracy. Write confirmation gate prevents unintended ledger modifications.

3 Scalable Blockchain-Based Traceability for Agricultural Imagery 2024 · Hyperledger Fabric · IPFS · Node.js

Applied the DV-TI infrastructure to citrus drone imagery (TIFF) from Immokalee, FL. Developed a dynamic hashing ID mechanism that updates when data is transformed (e.g., NDVI analysis, yield estimation), ensuring traceability without redundant on-chain storage. Tested with smart contract-based logging for uploads, transformations, downloads, and access events.

4 SI — Source Integration

Governs how files move between organizations. Access request workflow (Request → Approve → Fulfill → Download), on-chain transfer records, reuse evaluation with 7 Creative Commons license types, expiration enforcement, and decision chain tracing.

Sourced by: Cho, Y., Yu, Z., & Ampatzidis, Y. Under review.

5 TI — Transformation Integrity

After a file is produced from a registered input and sent to a downstream party, the downstream party can determine which registered input was used and what operation was declared — without contacting the producing environment. Prototype validated with a two-stage chain returning resolved, resolved with gap, or no declaration.

Sourced by: Cho, Y., Yu, Z., & Ampatzidis, Y. An Infrastructure for Determining the Declared Registered Input and Operation for a Transformed Agricultural File. Under review.

6 DV — File Origin Verification

After a file has been sent through multiple parties, a downstream receiver can determine whether the file has the same content as the version recorded before the first transfer, and reconstruct the sequence of senders and receivers — using only a shared external record, without contacting any earlier computer.

Sourced by: Cho, Y., Yu, Z., & Ampatzidis, Y. An Infrastructure for Agricultural File Origin Verification and Transfer Tracing. Under review.

7 Cross-Sectoral Review of Blockchain in Data Sharing

Analyzed blockchain applications in data sharing across agriculture, healthcare, and finance. Established the conceptual foundation for DV, TI, and SI as the three necessary layers for verifiable agricultural data exchange.

Sourced by: Cho, Y., Yu, Z., & Ampatzidis, Y. Facilitating a Future Agricultural Data Ecosystem. Under review.
